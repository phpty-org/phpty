("terminal" etc.) applies here, per the scope note in `CONTEXT-MAP.md`.
Surveyed at submodule commit `edf8d6b` (2026-06-18), gem version **0.6.3**
(`lib/reline/version.rb:2`). Gemspec declares `io-console ~> 0.5` as the only

## 2. Responsibilities and dependency graph

- **`Reline` / `Reline::Core`** (`reline.rb`) — top of the stack. Owns one `Config`, one `KeyStroke`, one `LineEditor`, delegates to `IOGate` (chosen once at require time, `reline.rb:521`). Drives the read loop (`read_io`, `reline.rb:378-406`) and calls `line_editor.update(key)` / `rerender`.
- **`LineEditor`** (2384 lines) — the largest class. Depends on `Unicode`, `Config`, `KillRing`, `Face`, and `IOGate` (all screen writes). Internally: state/lifecycle, rendering (`render`, `render_differential`, `render_line_differential`, dialog classes), input dispatch (`update`, `input_key`, `wrap_method_call`), and ~200 `private def` editing commands (`ed_*`, `em_*`, `vi_*`) invoked purely by method-symbol lookup.
- **`KeyStroke`** — stateless matcher; depends only on `Config#key_bindings` and encoding. Pure input-classification logic.
- **`KeyActor::Base`** — pure trie/hash keymap; `Composite` layers several instances, selected by `Config#editing_mode_label`. The Emacs/Vi mapping files are pure static data.
- **`Unicode`** — pure functions, no instance state, depends only on the `EastAsianWidth` table.
- **`Config`** — inputrc parsing into keymaps + variables; no rendering dependency; one IOGate call for encoding.
- **`History`** — thin array wrapper, trims to `config.history_size`.
- **`KillRing`** — pure data structure, used only by emacs kill commands.
- **`Face`** — pure SGR sequence builder (truecolor/256-color), consumed by dialog rendering.
- **`IO` + `ANSI`/`Dumb`** — the only classes doing real terminal control.

Call direction is strictly one-way: `Core` → `LineEditor` → (`Unicode`, `Config`, `KillRing`, `Face`, `IOGate`); `KeyStroke`/`KeyActor` are consulted by `Core`/`Config`, never call back into `LineEditor`.

## 3. Layering (the seams)

- **Pure logic, zero IO** — `Unicode`, `EastAsianWidth`, `KeyStroke`, `KeyActor::*`, `KillRing`, `History`, `Face`, and the buffer-manipulation half of `LineEditor`. Directly portable to PHPUnit without any tty.
- **Config/inputrc parsing** — logic-only, touches filesystem/ENV; easy to isolate.
- **Terminal I/O** — `IO`/`ANSI`/`Dumb`: raw mode, DSR probing, screen-size, SIGWINCH, buffered writes. See `reline-io-contract.md`.
- **Completion/dialog UI** — inside `LineEditor` (`DialogProcScope`, `Dialog`, `update_dialogs`, `DEFAULT_DIALOG_PROC_AUTOCOMPLETE` at `reline.rb:211-247`); seam is the `DialogRenderInfo` struct (pure data) vs the paint step.
- **Rendering** — geometry/diff computation is pure logic on cell arrays; only the final IOGate calls touch the tty.

## 4. Rendering model

Not a full redraw — **incremental cell-diff rendering** against a cached `@rendered_screen`:

1. `render` (`line_editor.rb:473`) builds `new_lines`: each visible row is a list of `[x, width, content]` cell-tuples — prompt, input, optional rprompt, menu rows, dialog overlay rows — positioned via `dialog_range`.
2. `render_differential` (`line_editor.rb:488`) diffs rows against the previous frame; only changed rows get `move_cursor_down` + `render_line_differential`. Handles scrolling when content exceeds `screen_height` (`IOGate.scroll_down`, adjusts `base_y`).
3. `render_line_differential` (`line_editor.rb:406`) per-cell diff within a row: computes overlay levels (topmost visible cell per column, dialogs can cover the line), chunks columns into skip/blank/level spans, re-emits only changed spans. This is what makes autocomplete popups redraw without flicker.
4. Cursor repositioned last via `wrapped_cursor_position`.
5. `render_finished` (`line_editor.rb:461`) — the only full-flush path, at accept-line.
6. All writes wrapped in `IOGate.buffered_output`.

Wrapping computed by `wrapped_prompt_and_input_lines`/`split_line_by_width` (display width, not bytes).

## 5. Key input pipeline

`Reline::Core#read_io` (`reline.rb:378-406`):
1. Read one byte via `io_gate.getc(timeout)`; timeout is infinite until `KeyStroke#match_status` reports `MATCHING_MATCHED` (complete match that could extend, e.g. bare ESC); then `keyseq_timeout` (default 500ms) applies.
2. `KeyStroke#match_status` (`key_stroke.rb:22`) consults the keymap; unresolved ESC-led sequences go to `match_unknown_escape_sequence` (`key_stroke.rb:80`) which parses generic CSI (`ESC [ params intermediates final`) and SS3 (`ESC O char`) shapes; vi mode disallows bare `ESC char`.
3. On MATCHED/UNMATCHED, `KeyStroke#expand` yields `Reline::Key` structs (char + method symbol), pushes back unconsumed bytes via `ungetc`, and expands single-byte inputrc macros.
4. Each `Key` goes to `line_editor.update(key)` → dispatch by method symbol to the `ed_*`/`em_*`/`vi_*` method.
5. Keymaps come from `Config`: full inputrc syntax (`config.rb:122-378`; `KEYSEQ_PATTERN` at `config.rb:4`): `set var value`, `"keyseq": function`/macro, `$if`/`$else`/`$endif`, `\C-`/`\M-`/`\e`, octal/hex.

## 6. Unicode/width handling

- Table: generated `[codepoint_upper_bound, width]` pairs; widths 1, 2, or -1 (ambiguous).
- `Unicode.east_asian_width(ord)` (`unicode.rb:75-79`) binary-searches and maps `-1` → `Reline.ambiguous_width` (runtime-resolved).
- `get_mbchar_width` (`unicode.rb:81-102`) handles ZWJ (U+200D) for emoji sequences. `calculate_width` (`unicode.rb:104-128`) iterates grapheme clusters; with `allow_escape_code: true` a `WIDTH_SCANNER` regex also skips CSI/OSC and the `\1...\2` non-printing bracketing used in prompts.
- **Ambiguous-width probe** — `reline.rb:408-427` (`may_req_ambiguous_char_width`): skip (width 1) if not a tty or gate is dumb; else write `▽` (U+25BD) at column 0, query `cursor_pos.x`; landed at 2 → ambiguous width is 2. One-time, cached. The DSR query itself is `io/ansi.rb:186-211` (`\e[6n` → parse `\e[row;colR`).

## 7. Test layout (`test/reline/*`)

Portable pure-logic (straightforward PHPUnit ports): `test_unicode.rb` (30 tests), `test_key_stroke.rb`, `test_key_actor_emacs.rb` (121 tests), `test_key_actor_vi.rb` (drive `LineEditor#update` programmatically via `Reline::TestCase`), `test_config.rb`, `test_history.rb`, `test_kill_ring.rb`, `test_face.rb`, `test_ansi.rb`, `test_string_processing.rb`, `test_macro.rb`, `test_reline_key.rb`. `test_line_editor.rb` includes a `TestIO < Reline::IO` fake to unit-test `render_line_differential` in-memory — a good template for a PHP fake-gate double. `test_reline.rb` mostly drives via StringIO; a PTY-gated subset exists.

Needs a real terminal: `test/reline/yamatanooroti/test_rendering.rb` (1988 lines) — spawns a pty and asserts on rendered screen contents. **This is exactly what ScreenTest reimplements**; the PHP port's rendering tests go through ScreenTest.

`download-test_readline.rb` replays Ruby stdlib's Readline-compat suite — mine for the compat surface, don't port structurally.

## 8. Feature tiers for the incremental port

- **Tier 0 — Foundations**: `Unicode` + `EastAsianWidth`, `KeyStroke`, `KeyActor::Base`/`Composite` + `EMACS_MAPPING`. Deliverable: byte stream → `Key` pipeline, headless-testable.
- **Tier 1 — Minimal single-line editor**: `LineEditor` buffer state + minimal `ed_*`/`em_*` subset, minimal `Config`, an ANSI-gate equivalent, single-row rendering. Riskiest piece is the renderer port — record the approach in an ADR when starting it.
- **Tier 2 — Multiline + wrapping + scrolling**: `wrapped_prompt_and_input_lines`, `split_line_by_width`, scroll handling, `@buffer_of_lines`.
- **Tier 3 — History**: `History` + navigation/search commands. Parallelizable with Tier 2 once Tier 1 exists.
- **Tier 4 — Completion (dialog UI)**: dialog classes, autocomplete proc, overlay rendering branch. Depends on Tier 2.
- **Tier 5 — Vi mode**: both vi tables + ~80 `vi_*` methods. Independent of 3/4; deferrable furthest.
- **Tier 6 — Face/theming**: SGR builder; Tier 4 can ship with hardcoded colors first.
- **Tier 7 — Full inputrc**: pure parsing, no architectural risk; grow coverage last.
- **Tier 8 — Windows gate**: out of scope (ADR-0006).
